# API Документация (API Documentation)

В платформе используется REST-подобный API для взаимодействия фронтенда и бэкенда.

## Основные эндпоинты

### Аутентификация (`/api/auth`)
- `POST /auth/login`: Вход через Telegram данных (initData). Синхронизирует профиль пользователя с БД.
- `POST /auth/bus-login`: Вход для водителей автобусов по номеру телефона и паролю.

### Поездки (`/api/rides`)
- `GET /rides`: Получение списка активных поездок с фильтрацией по городам и дате.
- `POST /rides`: Создание новой поездки (водитель или пассажир).
- `GET /rides/:id`: Детальная информация о поездке.
- `POST /rides/:id/book`: Бронирование места в поездке.
- `DELETE /rides/:id/cancel-booking`: Отмена бронирования.

### Автобусы (`/api/bus-tickets`)
- `GET /bus-tickets`: Список доступных автобусных рейсов.
- `GET /bus-tickets/:id`: Детали рейса и схема мест.
- `POST /bus-tickets/book`: Бронирование мест на автобус.

### Администрирование (`/api/admin`)
- Данные разделы защищены проверкой пароля/роли.
- `GET /admin/stats`: Статистика для дашборда.
- `GET /admin/users`: Управление списком пользователей.
- `GET /admin/rides`: Полный список поездок платформы.

## Формат ответов

Все ответы возвращаются в формате JSON:

**Успех:**
```json
{
  "data": { ... }
}
```

**Ошибка:**
```json
{
  "error": "Описание ошибки"
}
```

## Безопасность

API использует Supabase JWT для авторизации запросов. Заголовок `Authorization: Bearer <token>` обязателен для большинства защищенных операций.
