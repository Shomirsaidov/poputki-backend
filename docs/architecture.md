# Архитектура бэкенда (Backend Architecture)

Этот документ описывает архитектуру серверной части платформы Poputki.online.

## Стек технологий

- **Node.js**: Среда выполнения JavaScript.
- **Express.js**: Веб-фреймворк для создания API.
- **Supabase**: Backend-as-a-Service (BaaS), используемый для базы данных (PostgreSQL), аутентификации и хранения файлов.
- **Node-Telegram-Bot-Api**: Библиотека для взаимодействия с Telegram Bot API.

## Структура проекта

Серверная часть организована по модульному принципу:

- `server.js`: Главный файл входа, инициализация Express и подключение маршрутов.
- `routes/`: Директория с маршрутами API, разделенными по функционалу:
  - `auth.js`: Аутентификация через Telegram.
  - `rides.js`: Управление поездками (создание, поиск, бронирование).
  - `bus-tickets.js`: Управление автобусными рейсами.
  - `admin.js`: Панель администратора.
  - `users.js`: Профили пользователей.
  - `general.js`: Общие данные (города, уведомления).
- `utils/`: Вспомогательные функции:
  - `telegramBot.js`: Логика работы с Telegram (уведомления, широковещательные рассылки).
  - `supabase.js`: Настройка клиента Supabase.
- `docs/`: Техническая документация (этот раздел).

## Поток данных (Data Flow)

1. **Запрос от клиента**: Мини-приложение (Telegram WebApp) отправляет HTTP-запросы к Express API.
2. **Аутентификация**: Большинство маршрутов проверяют заголовок `Authorization` (JWT от Supabase). Данные пользователя синхронизируются с Telegram при первом входе.
3. **Бизнес-логика**: Маршруты обрабатывают данные, взаимодействуют с Supabase через клиентскую библиотеку и отправляют уведомления в Telegram при необходимости.
4. **Уведомления**: При создании поездки или бронировании сервер использует `telegramBot.js` для отправки сообщений в каналы или лично пользователям.

## Развертывание

Бэкенд спроектирован для работы на Node.js сервере (например, VPS с PM2 или облачные платформы). База данных и хранилище находятся в Supabase.
